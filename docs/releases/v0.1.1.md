# spoolbook v0.1.1 — Release packaging fixes

**Released**: 2026-07-14

---

## Highlights

- **macOS release fix**: release builds now package as a real `Spoolbook.app` bundle instead of a loose folder of files — a bare `dotnet publish` output has no Dock icon or double-click launch, since macOS reads those from `Contents/Info.plist`/`Contents/Resources/*.icns`, which only exist inside a proper app bundle.
- **macOS "is damaged" fix**: the app bundle is now ad-hoc signed. An entirely unsigned bundle downloaded from the internet fails Gatekeeper's basic check outright ("is damaged and can't be opened"); ad-hoc signing gets it to the normal "unidentified developer" right-click-Open prompt instead. Full notarization (removing that prompt too) would need a paid Apple Developer account — not done here.
- **CI now verifies each release build actually runs on its target OS**: win-x64 was being cross-published from a Linux runner, which only proves the build compiles, not that it launches on Windows. Every release leg now builds on its real target OS and gets smoke-tested (launch, wait 5s, confirm it didn't crash) before being uploaded.
- Repo now enforces a feature-branch + PR workflow on `main` (GitHub branch protection + a local pre-push guard) — no more direct pushes.

## Upgrading

No data/schema changes. Existing installs pick up nothing automatically since this is a packaging/CI fix, not an app update mechanism — download the new release when convenient.
