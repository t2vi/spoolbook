# spoolbook v0.1.2 — Printer and Project entities

**Released**: 2026-07-15

---

## Highlights

- **Printer is now a first-class entity**: previously a free-text string on Print, now a named, unique entity (with an optional model) managed under Settings → Printers. Existing free-text values are dropped on upgrade — see Upgrading below.
- **Attach a `.3mf` project file to a Print**: a Print can now reference the actual sliced project that was sent to the printer, alongside its Profile/Spool/Printer. Linked by path rather than copied in (project files can be tens of MBs) — pick an existing one from a dropdown or browse for a new file directly from the Print form. If the linked file moves, is deleted, or changes on disk, a non-blocking badge flags it (checked via a cheap mtime/size stat, not a content hash).

## Upgrading

Schema migration runs automatically on next launch (`db.Database.Migrate()`). One breaking data change: `Print.Printer` (free-text) is replaced by a `PrinterId` foreign key — existing Prints lose their printer text and need to be re-linked to a `Printer` record (add one under Settings → Printers, then edit each existing Print to select it). No other data loss; the new Project link is purely additive and optional.
